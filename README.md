# Qiekj Android

这是一个将 [`3ryng1um/qiekj`](https://github.com/3ryng1um/qiekj) 相关 Python 脚本流程改造为 Android App 的项目。当前 App 使用 Kotlin、Jetpack Compose、Retrofit 和 OkHttp 实现。

> 说明：本项目用于复刻个人脚本的请求流程和移动端界面。请自行承担账号、设备、接口变更和平台规则风险。不要将个人 Token、抓包文件或签名密钥上传到公开仓库。

## 当前功能

- 设备控制：读取历史设备，按原脚本流程执行设备启动、解锁、同步和订单详情查询。
- 积分任务：执行签到、首页浏览、任务列表、视频任务、支付宝视频任务和积分统计等自动化流程，并实时显示日志。
- 我的页面：支持手机号验证码登录、本地 Token 加密保存、资产查看、当前 Token 查看和历史订单查看。
- 本地历史：成功查询到订单详情后保存到本地，方便后续查看最近订单。
- 后台处理：网络请求在 IO 线程执行，等待逻辑使用协程 delay，避免阻塞界面。

## 使用说明

### 登录

1. 打开 App，进入底部导航的“我的”。
2. 输入手机号并发送验证码。
3. 输入短信验证码后确认登录。
4. 登录成功后，App 会加密保存 Token，后续冷启动会自动复用。

### 启动设备

1. 进入“设备控制”。
2. 等待历史设备加载完成。
3. 点击目标设备并等待流程执行结束。
4. 设备使用结束后，App 会展示订单详情并保存到历史订单。

### 执行积分任务

1. 先在“我的”页面完成登录。
2. 进入“积分任务”。
3. 点击“开始执行自动化任务”。
4. 在日志窗口查看实时执行结果。

### 查看历史订单

1. 进入“我的”。
2. 点击“历史订单”。
3. 在弹出的页面中查看最近保存的订单。

## 注意事项

- Token 会随手机号重新登录而变化，App 会保存最近一次登录得到的 Token。
- 如果积分任务在 Python 脚本中正常、但在 App 中失败，优先查看日志中的 HTTP 状态码、响应解析错误和 Token 前后 8 位。
- 接口返回结构变化时，可能需要同步更新数据模型和请求解析逻辑。
